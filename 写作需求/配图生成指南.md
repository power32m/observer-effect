# 科幻小说配图生成规则

## 一、图片规格（必须严格遵守）

| 参数 | 值 | 说明 |
|------|-----|------|
| **分辨率** | **1280×720** | 16:9 横幅，统一格式 |
| **格式** | JPG | |
| **宽高比** | 16:9 | 横幅，禁止生成方图或竖图 |
| **存放路径** | `E:\Opencode\科幻小说\images\` | |
| **命名规则** | `ep{集}{章}.jpg` | 例：`ep0103.jpg` |

**⚠️ 重要：生成后必须用 PIL 检查分辨率是否为 1280×720，不符合必须重生成。**

---

## 二、图片生成流程（必须按顺序执行）

```
1. 写完章节内容
   ↓
2. 根据剧情设计场景描述
   ↓
3. 按提示词模板组合 prompt，加入"16:9"和"1280x720"说明
   ↓
4. 调用 image_generate，model=minimax/image-01，size=1280x720
   ↓
5. 用 PIL 检查分辨率（必须为 1280×720）
   ↓
6. 保存到 E:\Opencode\科幻小说\images\
   ↓
7. 用 Start-Process 打开图片让用户查看
   ↓
8. git add + commit + push
```

## 三、生成命令

```python
# 1. 生成图片（指定 size=1280x720）

from PIL import Image
```

## 四、提示词模板（科学家手绘笔记风格）

### 基础模板
```
科学家手绘笔记风格，铅笔线条画，[具体场景描述]，手写英文字母涂鸦边缘，潦草的物理公式和图表，yellowed paper texture，细腻铅笔阴影，白底，无黑边，无边框，16:9 cinematic composition
```

### 必须包含的关键词
- `白底` / `white background`
- `无黑边` / `no border`
- `无边框` / `clean edge`
- `科学家手绘笔记风格`
- `16:9` / `wide aspect ratio`
- `1280x720`（在 prompt 中注明）

### ⚠️ 禁止出现的元素
- 任何**中文字符**
- **黑边/白边**（图片四边必须干净）
- 方图 1:1 或竖图 9:16

## 六、分辨率检查代码

```python
from PIL import Image
import os

def check_resolution(path):
    img = Image.open(path)
    if img.size != (1280, 720):
        print(f"⚠️ 分辨率错误: {img.size[0]}x{img.size[1]}，应为 1280x720")
        return False
    print(f"✅ 分辨率正确: {img.size[0]}x{img.size[1]}")
    return True

# 检查所有配图
base = r"E:\Opencode\科幻小说\images"
for f in os.listdir(base):
    if f.endswith('.jpg'):
        check_resolution(os.path.join(base, f))
```

## 八、常见问题

| 问题 | 解决 |
|------|------|
| 分辨率是 1024×1024 | 重新生成，指定 `size="1280x720"` |
| 图片有黑边 | prompt 加 `白底，无黑边，无边框` |
| 出现中文乱码 | prompt 中禁止出现任何中文字符 |
| 风格不统一 | 统一用"科学家手绘笔记风格"为基础 |
