# 科幻小说配图生成规则

## 风格A：科学家手绘笔记 (Scientist Handwritten Notes)

### ⚠️ 重要规则（必须遵守）

1. **不要出现中文** — 图片中不要包含任何中文字符
2. **不要出现黑边** — 确保图片四边干净，无黑边白边
3. **底色要与阅读器背景搭配** — 阅读器底色为 `#f5f0e6`，图片底色应使用浅灰米色/淡黄色，营造统一感
4. **推荐色值** — 纸张底色使用 `#ebe5d8` 或 `#f5f0e6` 系列

### 🎨 推荐配色

| 元素 | 色值/关键词 |
|------|------------|
| 纸张底色 | #ebe5d8, #f5f0e6, 浅灰米色 |
| 铅笔线条 | 灰色/浅棕色，柔和不刺眼 |
| 整体色调 | 淡雅、温暖、与阅读器协调 |

---

### 核心提示词模板

```
科学家手绘笔记风格，铅笔线条画，[具体场景描述]，手写英文字母涂鸦边缘，潦草的物理公式和图表，黄色纸张质感，细腻的铅笔阴影，白底，无黑边
```

### ⚠️ 禁止出现的元素

| 禁止 | 说明 |
|------|------|
| 中文字符 | 绝对不要在图片中出现任何中文 |
| 黑边/白边 | 图片四边必须干净，无边框 |
| 深色背景 | 使用浅色/白色背景 |

### ✅ 正确示例

**EP01-01 量子隧穿 - 配图提示词（已验证效果良好）：**
```
科学家手绘笔记风格，铅笔线条画，未来太空舱内场景，地球悬浮在舷窗外，太空人凝视窗外，身上连接着发光的量子线缆，背景有深蓝色的星空，手写英文字母涂鸦，边缘潦草物理公式，黄色纸张质感，细腻铅笔阴影，白底，无黑边，无边框
```

### 成功经验
- 关键词 "白底，无黑边，无边框" 配合使用效果最佳
- 纸张质感用 "yellowed paper texture" 配合淡色底
- 整体效果：淡雅、温暖、与阅读器背景(#f5f0e6)协调

### 关键词库

| 元素 | 英文关键词 |
|------|-----------|
| 铅笔线条 | pencil sketch, graphite pencil |
| 手写字体（英文） | handwritten notes, cursive script |
| 涂鸦/注释 | margin doodles, annotation |
| 科学公式 | scientific formulas, equations |
| 纸张质感 | yellowed paper texture, aged paper |
| 无黑边 | no border, clean edge, white margin |
| 白底 | white background, light background |

### 图片生成参数建议

- **aspectRatio**: 16:9 或 3:2
- **model**: minimax/image-01
- **prompt**: 使用上述模板 + 具体场景描述

### 存储位置

生成的配图保存在：`E:\Opencode\科幻小说\配图\`

命名规则：`epXX-XX.jpg`（如 `ep01-01.jpg`）

## 生成图片的操作记录

### 标准化操作流程

```
1. 根据章节内容设计场景描述
   ↓
2. 按照提示词模板组合
   ↓
3. 调用 image_generate 生成
   ↓
4. 保存到配图文件夹
   ↓
5. 用 Start-Process 打开图片让用户查看
   ↓
6. 更新 reader.html 中的图片路径
```

### 常用命令

```powershell
# 复制并打开图片
Copy-Item "<源路径>" "E:\Opencode\科幻小说\配图\<目标文件名>"
Start-Process "E:\Opencode\科幻小说\配图\<目标文件名>"

# 打开阅读器
Start-Process "E:\Opencode\科幻小说\reader.html"
```

### 常见问题

| 问题 | 解决 |
|------|------|
| 图片有黑边 | 提示词加"白底，无黑边，无边框" |
| 中文乱码 | 提示词不要出现中文字符 |
| 风格不统一 | 统一用"科学家手绘笔记风格"为基础 |

## 生成流程

1. 根据章节剧情设计场景描述
2. 按照模板组合提示词
3. 确保包含"无黑边""白底"等关键词
4. 生成后检查是否有黑边
5. 如有黑边，重新生成或裁剪边缘
