# 科幻小说阅读器 - 项目规范

## 一、项目结构

```
observer-effect/
├── index.html          # 阅读器主页面（纯前端，无需构建）
├── chapters/          # 章节源文件（Markdown 格式）
│   ├── EP01-01.md    # 第1集第1章
│   ├── EP01-02.md    # 第1集第2章
│   └── ...
├── images/            # 配图（与 chapters 同级）
│   ├── cover.jpg      # 封面图
│   ├── ep01-01.jpg   # 第1章配图
│   ├── ep01-02.jpg   # 第2章配图
│   └── ...
```

**规则：所有文件夹和文件统一用英文命名，章节文件用 `EP{集:2位}-{章:2位}.md` 格式。**

---

## 二、阅读器功能需求

### 2.1 页面结构

- **顶部 Header**：固定定位，小说标题 + 副标题，z-index 100
- **左侧 Sidebar**：章节目录导航，固定宽度 220px，sticky 定位，**自身内部可滚动**（章节再多也不会截断）
- **右侧 Content**：封面页或章节正文
- **无底部导航栏**，纯桌面端设计

### 2.2 封面页

- 显示封面图片（点击进入第1章）
- 显示小说标题、作者（无痛穿耳）、标签
- "开始阅读"按钮
- 封面页**不显示**左侧 sidebar

### 2.3 章节页

- 顶部配图（宽度100%）
- 配图下方章节正文
- 底部有"上一章 ←" / "→ 下一章"翻页按钮
- 左侧 sidebar 显示章节目录，当前章节高亮
- **加载失败时显示友好提示**，不白屏

### 2.4 章节切换

- sidebar 中"◀ 封面" → 返回封面
- sidebar 中章节标题 → 进入该章节
- 正文底部翻页按钮 → 上一章/下一章
- 当前章节在 sidebar 中高亮

## 三、章节文件格式

### 3.1 文件命名

格式：`EP{集数:2位}-{章数:2位}.md`
示例：`EP01-01.md`、`EP01-02.md`、`EP02-01.md`

### 3.2 内容格式

```markdown
# EP01-01：章节标题

**【时间】** 2075年3月15日 03:47 | **【地点】** 地球轨道观测站「拉格朗日-3」

---

正文内容...

---

正文继续...
```

### 3.3 配图命名规则

- 封面：`images/cover.jpg`
- 各章配图：`images/ep{集:2位}{章:2位}.jpg`
- 例：`images/ep0101.jpg`、`images/ep0102.jpg`
- 支持格式：JPG 和 PNG

## 四、技术实现要求

### 4.1 阅读器架构

- **纯前端**：HTML + CSS + JavaScript，无后端依赖
- **Markdown 解析**：使用 [marked.js](https://cdn.jsdelivr.net/npm/marked/marked.min.js)，引入 CDN 即可
- **章节加载**：`fetch()` 加载 `.md` 文件，解析后渲染

### 4.2 章节路径（重要！）

使用**相对路径**，本地和线上都能正常工作：

```javascript
// ✅ 正确：相对路径，本地 python -m http.server 和 GitHub Pages 都能用
var CHAPTER_ORDER = [
    "chapters/EP01-01.md",
    "chapters/EP01-02.md",
];

// ❌ 错误：绝对 URL，本地测试时 fetch 不到本地修改的文件
// var CHAPTER_ORDER = [
//     "https://power32m.github.io/observer-effect/chapters/EP01-01.md",
// ];
```

### 4.3 配图路径推导

**不需要 IMAGE_MAP**，用函数自动推导：

```javascript
function getImagePath(chapterId) {
    // "EP01-01" → "images/ep0101.jpg"
    var num = chapterId.replace("EP", "").replace("-", "");
    return "images/ep" + num.toLowerCase() + ".jpg";
}
```

新增章节**只需在 CHAPTER_ORDER 添加一条**，其他全部自动。

### 4.4 marked.js 使用方式

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
<script>
    // 配置 marked
    marked.setOptions({
        breaks: true,    // 换行符转为 <br>
        gfm: true        // GitHub Flavored Markdown
    });

    // 解析 Markdown
    var html = marked.parse(markdownText);
    document.getElementById("content").innerHTML = html;
</script>
```

### 4.5 样式规范

| 项目 | 值 |
|------|-----|
| 字体 | Noto Serif SC（Google Fonts） |
| 背景色 | #f5f0e6 |
| 正文 | 1.1rem，行高 1.8 |
| 标题色 | #2c2c2c |
| 正文色 | #3c3c3c |
| Sidebar 宽 | 220px |
| Sidebar 自身滚动 | `overflow-y: auto` + `height: calc(100vh - header高度)` |
| Header | `position: fixed`，z-index 100 |
| 内容区最大宽 | 1200px，水平居中 |

Sidebar CSS 示例：
```css
.sidebar {
    position: sticky;
    top: 60px;               /* header高度 */
    width: 220px;
    height: calc(100vh - 60px);
    overflow-y: auto;         /* 内部滚动，章节再多也不截断 */
    flex-shrink: 0;
}
```

### 4.6 错误处理

`fetch()` 失败时显示友好提示：
```javascript
.catch(function(err) {
    document.getElementById("contentArea").innerHTML =
        '<div style="text-align:center;padding:60px;color:#8b7355">' +
        '<p>章节加载失败，请检查网络后刷新页面。</p>' +
        '<p style="font-size:0.8rem;margin-top:10px">' + path + '</p>' +
        '</div>';
});
```

## 五、工作流程

### 5.1 新增章节（只需改一处！）

1. 在 `chapters/` 创建 `EP01-03.md`
2. 在 `images/` 放入 `ep0103.jpg`（配图）
3. 在 `index.html` 的 `CHAPTER_ORDER` 数组末尾添加：
   ```javascript
   "chapters/EP01-03.md",
   ```
4. 提交 GitHub，完成

### 5.2 本地测试

```bash
cd /path/to/observer-effect
python -m http.server 8080
# 浏览器打开 http://localhost:8080
```

修改 `.md` 文件后，刷新浏览器即可看到最新内容。
